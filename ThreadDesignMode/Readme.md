## 总结：
### 1. 多线程程序好不好评判的标准
>1.2两点是必要的，3.4两点是锦上添花
1. 安全性
    - **保证只是状态变了**。
    - 不损坏对象，通常是指对象的属性出现了意想不到的情况。
    **例如：i--, 在没有保护的情况下就有可能出现问题**
    - 安全性和兼容性
        - arrayList如何安全？？
    
2. 生存性
    - **状态没有变，但是程序停了**，或者是即使程序没有死，但是也是不动，卡死了（死锁）
    - 必要的处理能够被执行，出现假死，最典型的是死锁。
    - 生存性和安全性是相互制约的，一味强调安全性，那么就有可能出现生存性问题。
3. 可重复性
    - juc里面其实有大量可重复利用的类
4. 性能
    - doug lea 总结了影响性能的关键因素，如下
      1. 吞吐量
      2. 响应性
      3. 容量
